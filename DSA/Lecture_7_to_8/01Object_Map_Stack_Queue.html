üß± Objects in JavaScript

Objects are collections of key-value pairs. They are used to store and organize data in JavaScript.

const person = {
  name: "Rahul",
  age: 25,
  isStudent: true
};

You can access properties using dot or bracket notation:

console.log(person.name);      // Output: "Rahul"
console.log(person["age"]);    // Output: 25
Why use Objects?
  • Organize related data
  • Store functions (methods)
  • Flexible and dynamic

üó∫Ô∏è Maps in JavaScript

Maps are similar to objects, but they allow any type of key and maintain the insertion order.

const myMap = new Map();
myMap.set("name", "Amit");
myMap.set(1, "One");
myMap.set(true, "Boolean key");

To access values:

console.log(myMap.get("name"));  // Output: "Amit"
console.log(myMap.has(1));       // Output: true
Advantages of Maps:
  • Any data type as key
  • Preserves insertion order
  • Built-in methods like .set(), .get(), .has()

üìö Stacks - LIFO (Last In First Out)

A Stack is a linear structure where the last element added is the first one removed.

const stack = [];
stack.push(10);
stack.push(20);
console.log(stack.pop());  // Output: 20
Real-World Examples:
  • Undo feature in editors
  • Function call stack

üö∂ Queues - FIFO (First In First Out)

A Queue is a structure where the first element added is the first one removed.

const queue = [];
queue.push(10);
queue.push(20);
console.log(queue.shift());  // Output: 10
Real-World Examples:
  • Task scheduling
  • Customer service line
  • Printing queue

üìå Summary

  • Objects: Simple key-value stores, great for structured data.
  • Maps: Powerful key-value structure with order and any key type.
  • Stacks: Use .push() and .pop(). Last in, first out.
  • Queues: Use .push() and .shift(). First in, first out.
